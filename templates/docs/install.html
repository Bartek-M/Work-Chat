Ustawienie aplikacji Work-Chat
Tutaj pokażemy wszystko co musi wiedzieć przyszły administrator Work-Chat
Plik konfiguracyjny
Ta aplikacja wymaga pliku .env, należy użyć .env.example by stworzyć pasujący plik konfiguracyjny, bez niego serwer nie będzie działał poprawnie.
przykładowy plik .env z wytłumaczeniem
SECRET_KEY = "KEY_HERE" Klucz to szyfrowania haseł
DEBUG = "True" Opcja debugowania
STATIC_URL = "assets/" Miejsce położenia assetów(css, zdjęcia, ikony, JS)

LDAP_SERVER = 'AD_name' Nazwa domeny active directory
LDAP_ADMIN = 'cn=admin_account_name,cn=users,dc=first_AD_part,dc=second_AD_part' Dane konta administrator
LDAP_ADMIN_PASSWORD = 'admin_password' Hasło do konta administratora
LDAP_SEARCH_BASE = 'dc=first_AD_part,dc=second_AD_part' Dane domeny active directory do szukania kont
Sposób rozumienia danych do odczytywania kont z active directory
'dc=first_AD_part' Odnosi się do jednej części nazwy domeny
Na przykład: przyklad.local zapiszemy jako 'dc=przyklad,dc=local'
Dłuższy przykład: przyklad.edu.local zapiszemy jako 'dc=przyklad,dc=edu,dc=local'
Konfiguracja z Docker'em
Do instalacji wszystkich pakietów należy użyć:
docker-compose build

Aby włączyć serwer:
docker-compose up

Wyłączyć serwer możemy za pomocą polecenia:
docker-compose down
Konfiguracja bez Dockera
Potrzebny sprzęt
Do wykorzystania Work-Chat potrzebny jest:
  1. Serwer
  2. Sieć komputerowa
  3. komputery klienckie
Na serwerze poza oczywistymi rzeczami jak zasilanie, potrzebny też jest jakikolwiek uniksowy system operacyjny
W samym systemie potrzebny jest
  • Python 3.11 lub wyżej
  • Node.js 16 lub wyżej
Instalacja
Jeśli powyższe wymagania zostały spełnione należy pobrać aplikacje Work-Chat jako zip, i rozpakować ją w wybranym miejscu.
Następnie w konsoli trzeba po kolei wpisać te komendy:
  1. pip3 install -r requirements.txt
  2. npm install -g typescript
  3. npm install
  4. sudo apt install libmagic1
  5. sudo apt install gunicorn
  6. npm run build
Wytłumaczenie poszczególnych komend:
  1. Przy użyciu pip3 instalują się wszystkie wymagane biblioteczki
  2. Przy użyciu npm instaluje sie typescript
  3. Przy użyciu npm instaluje pakiety do node_modules, które są ustalone w package.json i package-lock.json
  4. Przy użyciu apt instaluje sie gunicorn
  5. Przy użyciu npm budują się pakiet JS
Uruchomienie
Należy wpisać te komendę gunicorn backend.wsgi:application
W wyniku jakiś niejasności należy odnosić się do README.md, lub README.md w folderze docs, lub do GitHub'a pod tym linkiem
